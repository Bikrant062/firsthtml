Keys

Day 4

  • Key is a field (or combination of fields) in a table that is used to uniquely identify rows in that table. Keys play a crucial role in maintaining the integrity and efficiency of the database.

Different types of of keys are discussed below

  1. Super Key

    2. A super key in a Database Management System (DBMS) is a set of one or more columns (attributes) that can uniquely identify a record within a table. Essentially, a super key is any combination of columns that guarantees uniqueness for each row in the table. It can include more attributes than necessary for unique identification.

  2. Candidate Key

    3. A candidate key in a Database Management System (DBMS) is a minimal super key, meaning it is a set of one or more columns (attributes) that uniquely identify a record in a table, and no subset of these columns can uniquely identify a record on its own. In other words, a candidate key is a super key without any unnecessary attributes.

  3. Primary Key

    4. A primary key in a Database Management System (DBMS) is a special type of key used to uniquely identify each record in a table. It is a fundamental concept for ensuring data integrity, efficient data retrieval, and the establishment of relationships between tables.

  4. Alternate Key

    5. An alternate key in a Database Management System (DBMS) is a candidate key that is not chosen as the primary key of a table. Alternate keys are still capable of uniquely identifying records in a table, but they serve as secondary unique identifiers, whereas the primary key is the main unique identifier.

  5. Foreign Key

    6. A foreign key in a Database Management System (DBMS) is a field (or collection of fields) in one table that uniquely identifies a row of another table or the same table. The primary purpose of foreign keys is to ensure referential integrity of the data. In other words, foreign keys enforce a link between the data in two tables.

  6. Composite Key

    7. A composite key in a Database Management System (DBMS) is a key that consists of two or more columns (attributes) that together uniquely identify a record in a table. Composite keys are useful in situations where a single column is not sufficient to uniquely identify rows within a table.

  7. Artificial Key

    8. An artificial key, also known as a surrogate key, is a type of key used in a Database Management System (DBMS) to uniquely identify each record in a table. Unlike natural keys, which are derived from the actual data and have intrinsic meaning, artificial keys are generated and have no inherent meaning outside of uniquely identifying a record. They are usually numeric or alphanumeric values generated by the system.
