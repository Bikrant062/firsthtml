Relationships in DBMS

In database design, relationships define how data is connected or related between different tables. These relationships are crucial as they ensure data integrity and provide a structure for querying and manipulating data efficiently.

One-to-One (1:1) Relationship:

In a one-to-one relationship, one record in a table is related to exactly one record in another table. This relationship is less common in database design but is useful for dividing a table with many fields for clarity or security reasons.

Example: A table "Person" may have a one-to-one relationship with a table "ContactInfo," where each person has exactly one set of contact information.

One-to-Many (1:N) Relationship:

In a one-to-many relationship, one record in a table can be related to one or more records in another table. This is the most common type of relationship in relational databases.

Example: A table "Department" can have a one-to-many relationship with a table "Employee," where each department can have multiple employees, but each employee belongs to only one department.

Many-to-One (M:1) Relationship:

A many-to-one relationship is essentially the inverse of a one-to-many relationship. Many records in a table can be related to one record in another table.

Example: Multiple employees can belong to the same department, establishing a many-to-one relationship between the "Employee" table and the "Department" table.

Many-to-Many (M:N) Relationship:

In a many-to-many relationship, multiple records in one table can be related to multiple records in another table. To implement this in a relational database, a junction table (also known as an associative or intersection table) is used to connect the two tables.

Example: A table "Student" may have a many-to-many relationship with a table "Course." A junction table "Enrollment" would then connect students to courses, indicating which students are enrolled in which courses.
